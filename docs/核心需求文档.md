# Flutter Smooth Markdown 渲染器 - 核心需求文档

## 1. 项目概述

Flutter Smooth Markdown 是一个高效、流畅的 Markdown 渲染器，专为 Flutter 应用设计。核心目标是提供实时流式渲染能力，支持完整的 Markdown 语法，并保持优秀的性能表现。

## 2. 核心功能需求

### 2.1 流式渲染（Streaming Rendering）

**功能描述：**
- 支持逐字符/逐块接收 Markdown 内容并实时渲染
- 适用于 AI 对话、实时通信等场景
- 渲染过程平滑无卡顿

**技术要求：**
- 采用增量解析策略，只解析新增内容
- 使用 Stream/StreamBuilder 实现数据流处理
- 支持缓冲区机制，优化渲染性能
- 实现智能重排版，避免全局刷新

**性能指标：**
- 单次渲染延迟 < 16ms（保持 60fps）
- 支持每秒处理 1000+ 字符的流式输入
- 内存占用增长线性可控

## 3. 技术架构要求

### 3.1 解析器（Parser）

**职责：**
- 将 Markdown 文本解析为抽象语法树（AST）
- 支持增量解析
- 错误处理和容错机制

**技术选型：**
- 考虑使用 `markdown` package 或自研解析器
- 支持可扩展的语法规则
- 实现高效的正则表达式匹配

### 3.2 渲染引擎（Renderer）

**职责：**
- 将 AST 转换为 Flutter Widget
- 管理渲染状态
- 优化渲染性能

**技术实现：**
- 使用 Widget Tree 构建
- 实现 Widget 缓存和复用
- 支持自定义样式主题

### 3.3 流式处理器（Stream Processor）

**职责：**
- 处理输入数据流
- 缓冲和分块策略
- 触发增量渲染

**技术实现：**
- 基于 Dart Stream API
- 实现智能缓冲区
- 支持背压（backpressure）处理

## 4. 性能优化策略

### 4.1 渲染优化
- 使用 `RepaintBoundary` 隔离渲染区域
- 实现虚拟滚动（长文档）
- Widget 缓存和复用机制

### 4.2 解析优化
- 增量解析，只处理变化部分
- 并发解析（isolate）
- 解析结果缓存

### 4.3 内存优化
- 及时释放不可见内容
- 图片内存管理
- 防止内存泄漏

## 5. API 设计

### 5.1 基础 API

```dart
// 静态渲染
SmoothMarkdown(
  data: '# Hello Markdown',
  styleSheet: customStyleSheet,
  onTapLink: (url) => handleLink(url),
)

// 流式渲染
SmoothMarkdownStream(
  stream: markdownStream,
  styleSheet: customStyleSheet,
  bufferSize: 256,
)
```

### 5.2 配置 API

```dart
MarkdownConfig(
  enableHtml: true,
  enableCodeHighlight: true,
  enableImageCache: true,
  syntaxHighlightTheme: 'github',
)
```

### 5.3 自定义渲染器

```dart
MarkdownBuilder(
  builders: {
    'code': CustomCodeBlockBuilder(),
    'image': CustomImageBuilder(),
  },
)
```
