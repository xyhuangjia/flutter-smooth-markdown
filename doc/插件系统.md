# 插件系统 (Plugin System)

Flutter Smooth Markdown 提供了一个灵活的插件系统，允许你扩展 Markdown 解析器以支持自定义语法。

## 概述

插件系统支持两种类型的插件：

1. **Block Parser Plugins** - 解析块级元素（如警告框、图表容器等）
2. **Inline Parser Plugins** - 解析行内元素（如 @提及、#标签、表情符号等）

## 快速开始

### 1. 创建插件注册表

```dart
import 'package:flutter_smooth_markdown/flutter_smooth_markdown.dart';

final registry = ParserPluginRegistry();
```

### 2. 注册内置插件

```dart
// 注册单个插件
registry.register(const MentionPlugin());
registry.register(const HashtagPlugin());
registry.register(const EmojiPlugin());
registry.register(const AdmonitionPlugin());

// 或批量注册
registry.registerAll([
  const MentionPlugin(),
  const HashtagPlugin(),
  const EmojiPlugin(),
  const AdmonitionPlugin(),
]);
```

### 3. 创建带插件的解析器

```dart
final parser = MarkdownParser(plugins: registry);
final nodes = parser.parse('Hello @john :wave: #flutter');
```

## 内置插件

### MentionPlugin (@提及)

解析 `@username` 语法，生成 `MentionNode`。

```dart
registry.register(const MentionPlugin());

// 输入: "Hello @john!"
// 输出: MentionNode(username: 'john')
```

**用户名规则**:
- 必须以字母开头
- 可以包含字母、数字、下划线和连字符

### HashtagPlugin (#标签)

解析 `#tag` 语法，生成 `HashtagNode`。

```dart
registry.register(const HashtagPlugin());

// 输入: "Check out #flutter!"
// 输出: HashtagNode(tag: 'flutter')
```

**标签规则**:
- 必须以字母或下划线开头
- 可以包含字母、数字和下划线

### EmojiPlugin (表情符号)

解析 `:shortcode:` 语法，生成 `EmojiNode`。

```dart
registry.register(const EmojiPlugin());

// 输入: "Hello :smile:!"
// 输出: EmojiNode(shortcode: 'smile', emoji: '😄')
```

**支持的表情符号**:
- 笑脸: smile, grinning, joy, wink, blush 等
- 手势: thumbsup, wave, clap, pray 等
- 心形: heart, broken_heart, sparkling_heart 等
- 动物: dog, cat, unicorn, dragon 等
- 更多...

**自定义表情符号**:
```dart
registry.register(const EmojiPlugin(
  customEmojis: {
    'custom': '🎉',
    'party': '🥳',
  },
));
```

### AdmonitionPlugin (警告框/提示框)

解析警告框语法，生成 `AdmonitionNode`。

```dart
registry.register(const AdmonitionPlugin());
```

**语法**:
```markdown
::: note
这是一个备注。
:::

::: warning 警告标题
请注意这一点！
:::

::: tip 提示
专业提示内容。
:::

::: danger 危险
危险操作警告！
:::
```

**支持的类型**:
- `note` / `info` - 备注/信息
- `tip` / `hint` - 提示
- `warning` / `caution` - 警告
- `danger` / `error` - 危险/错误
- `important` - 重要
- 其他 - 自定义类型

## 创建自定义插件

### 创建 Inline 插件

```dart
import 'package:flutter_smooth_markdown/flutter_smooth_markdown.dart';

// 1. 定义 AST 节点
class MyCustomNode extends MarkdownNode {
  const MyCustomNode(this.value);

  final String value;

  @override
  String get type => 'my_custom';

  @override
  Map<String, dynamic> toJson() => {
    'type': type,
    'value': value,
  };

  @override
  MyCustomNode copyWith({String? value}) {
    return MyCustomNode(value ?? this.value);
  }
}

// 2. 实现插件
class MyCustomPlugin extends InlineParserPlugin {
  const MyCustomPlugin();

  @override
  String get id => 'my_custom';

  @override
  String get name => 'My Custom Plugin';

  @override
  String get triggerCharacter => '~'; // 触发字符

  @override
  int get priority => 10; // 优先级（越高越先检查）

  @override
  bool canParse(String text, int index) {
    // 检查是否可以开始解析
    return text[index] == '~' &&
           index + 1 < text.length &&
           text[index + 1] != '~';
  }

  @override
  InlineParseResult? parse(String text, int startIndex) {
    // 解析逻辑
    var i = startIndex + 1;
    while (i < text.length && text[i] != '~') {
      i++;
    }

    if (i >= text.length) return null;

    final value = text.substring(startIndex + 1, i);
    return InlineParseResult(
      node: MyCustomNode(value),
      consumed: i - startIndex + 1,
    );
  }
}
```

### 创建 Block 插件

```dart
// 1. 定义 AST 节点
class DiagramNode extends MarkdownNode {
  const DiagramNode({
    required this.diagramType,
    required this.content,
  });

  final String diagramType;
  final String content;

  @override
  String get type => 'diagram';

  @override
  Map<String, dynamic> toJson() => {
    'type': type,
    'diagramType': diagramType,
    'content': content,
  };

  @override
  DiagramNode copyWith({String? diagramType, String? content}) {
    return DiagramNode(
      diagramType: diagramType ?? this.diagramType,
      content: content ?? this.content,
    );
  }
}

// 2. 实现插件
class DiagramPlugin extends BlockParserPlugin {
  const DiagramPlugin();

  @override
  String get id => 'diagram';

  @override
  String get name => 'Diagram Plugin';

  @override
  int get priority => 10;

  static final _startPattern = RegExp(r'^```(mermaid|plantuml)$');

  @override
  bool canParse(String line, List<String> lines, int index) {
    return _startPattern.hasMatch(line.trim());
  }

  @override
  BlockParseResult? parse(List<String> lines, int startIndex) {
    final firstLine = lines[startIndex].trim();
    final match = _startPattern.firstMatch(firstLine);
    if (match == null) return null;

    final diagramType = match.group(1)!;
    final contentLines = <String>[];
    var i = startIndex + 1;

    // 收集内容直到 ```
    while (i < lines.length) {
      if (lines[i].trim() == '```') {
        break;
      }
      contentLines.add(lines[i]);
      i++;
    }

    return BlockParseResult(
      node: DiagramNode(
        diagramType: diagramType,
        content: contentLines.join('\n'),
      ),
      linesConsumed: i - startIndex + 1,
    );
  }
}
```

## 插件注册表 API

### 注册方法

```dart
// 注册块级插件
registry.registerBlock(myBlockPlugin);

// 注册行内插件
registry.registerInline(myInlinePlugin);

// 自动检测类型注册
registry.register(plugin);

// 批量注册
registry.registerAll([plugin1, plugin2]);
```

### 查询方法

```dart
// 获取所有块级插件
List<BlockParserPlugin> blockPlugins = registry.blockPlugins;

// 获取所有行内插件
List<InlineParserPlugin> inlinePlugins = registry.inlinePlugins;

// 按 ID 获取插件
final plugin = registry.getBlockPlugin('my_plugin');
final inlinePlugin = registry.getInlinePlugin('mention');

// 按触发字符获取插件
final mentionPlugin = registry.getInlinePluginByTrigger('@');

// 检查触发字符
bool isATrigger = registry.isInlineTrigger('@');
```

### 管理方法

```dart
// 取消注册
registry.unregisterBlock('my_plugin');
registry.unregisterInline('mention');

// 清除所有插件
registry.clear();

// 复制注册表
final copy = registry.copy();
```

## 优先级

插件按优先级排序，数值越高越先被检查：

- 内置 Markdown 语法优先级为负数（-10 到 -1）
- 默认插件优先级为 0
- 自定义插件建议使用正数（1-100）

```dart
class HighPriorityPlugin extends InlineParserPlugin {
  @override
  int get priority => 100; // 最先检查
}
```

## 与渲染器集成

自定义插件生成的节点需要配套的 Builder 来渲染：

```dart
// 1. 创建 Builder
class MentionBuilder extends MarkdownWidgetBuilder {
  @override
  bool canBuild(MarkdownNode node) => node is MentionNode;

  @override
  Widget build(
    MarkdownNode node,
    MarkdownStyleSheet styleSheet,
    MarkdownRenderContext context,
  ) {
    final mention = node as MentionNode;
    return GestureDetector(
      onTap: () => _onMentionTap(mention.username),
      child: Text(
        '@${mention.username}',
        style: TextStyle(
          color: Colors.blue,
          fontWeight: FontWeight.bold,
        ),
      ),
    );
  }
}

// 2. 注册 Builder
final renderer = MarkdownRenderer();
renderer.registerBuilder('mention', MentionBuilder());

// 3. 使用
SmoothMarkdown(
  data: 'Hello @john!',
  parser: parser,  // 带插件的解析器
  renderer: renderer,  // 带自定义 Builder 的渲染器
)
```

## 最佳实践

1. **唯一 ID** - 确保每个插件有唯一的 ID
2. **合理优先级** - 根据语法冲突可能性设置优先级
3. **高效解析** - `canParse` 应该快速返回，详细检查放在 `parse` 中
4. **错误处理** - `parse` 返回 null 表示解析失败，会回退到其他解析器
5. **不可变节点** - AST 节点应该是不可变的，提供 `copyWith` 方法
