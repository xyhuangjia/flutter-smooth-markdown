## 1. 架构概览

Flutter Smooth Markdown 采用分层架构设计，主要分为以下几层：

```
┌─────────────────────────────────────────────┐
│         Widget Layer (UI层)                 │
│  - SmoothMarkdown                           │
│  - SmoothMarkdownStream                     │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│         Renderer Layer (渲染层)             │
│  - MarkdownRenderer                         │
│  - WidgetBuilder Framework                  │
│  - Element Builders                         │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│         Parser Layer (解析层)               │
│  - MarkdownParser                           │
│  - BlockParser                              │
│  - InlineParser                             │
│  - AST (Abstract Syntax Tree)               │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│       Stream Layer (流处理层)               │
│  - StreamProcessor                          │
│  - IncrementalParser                        │
│  - BufferManager                            │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│      Configuration Layer (配置层)           │
│  - MarkdownConfig                           │
│  - MarkdownStyleSheet                       │
└─────────────────────────────────────────────┘
```

## 3. 核心模块详解

### 3.1 AST (抽象语法树)

**位置：** `lib/src/parser/ast/markdown_node.dart`

**状态：** ✅ 已完成

**设计说明：**

AST 是 Markdown 文档的内存表示，所有节点继承自 `MarkdownNode` 基类。

**已实现的节点类型：**

| 节点类型 | 类名 | 用途 |
|---------|------|------|
| 文本 | `TextNode` | 纯文本内容 |
| 标题 | `HeaderNode` | H1-H6 标题 |
| 段落 | `ParagraphNode` | 段落容器 |
| 代码块 | `CodeBlockNode` | 代码块，支持语言标识 |
| 列表 | `ListNode` | 有序/无序列表 |
| 列表项 | `ListItemNode` | 列表项，支持任务列表 |
| 引用块 | `BlockquoteNode` | 引用内容 |
| 水平线 | `HorizontalRuleNode` | 分割线 |
| 行内代码 | `InlineCodeNode` | 行内代码 |
| 粗体 | `BoldNode` | 粗体文本 |
| 斜体 | `ItalicNode` | 斜体文本 |
| 删除线 | `StrikethroughNode` | 删除线文本 |
| 链接 | `LinkNode` | 超链接 |
| 图片 | `ImageNode` | 图片 |

**节点接口：**

```dart
abstract class MarkdownNode {
  String get type;                    // 节点类型标识
  Map<String, dynamic> toJson();      // JSON序列化
  MarkdownNode copyWith();            // 创建副本
}
```

### 3.2 配置系统

#### 3.2.1 MarkdownConfig

**位置：** `lib/src/config/markdown_config.dart`

**状态：** ✅ 已完成

**功能：**
- 控制功能开关（HTML、代码高亮、缓存等）
- 配置语法扩展（任务列表、表格、Emoji等）
- 自定义渲染行为

**主要配置项：**

```dart
MarkdownConfig(
  enableHtml: false,              // HTML标签
  enableCodeHighlight: true,      // 代码高亮
  enableImageCache: true,         // 图片缓存
  enableTaskLists: true,          // 任务列表
  enableStrikethrough: true,      // 删除线
  enableTables: true,             // 表格
  enableAutoLinks: true,          // 自动链接
  enableEmoji: false,             // Emoji
  enableLatex: false,             // LaTeX公式
  syntaxHighlightTheme: 'github', // 代码主题
)
```

#### 3.2.2 MarkdownStyleSheet

**位置：** `lib/src/config/style_sheet.dart`

**状态：** ✅ 已完成

**功能：**
- 定义所有Markdown元素的视觉样式
- 提供预设的亮色/暗色主题
- 支持完全自定义

**预设主题：**
- `MarkdownStyleSheet.light()` - 亮色主题
- `MarkdownStyleSheet.dark()` - 暗色主题

**可自定义的样式：**
- 文本样式（标题、段落、引用等）
- 装饰样式（边框、背景、圆角等）
- 间距布局（块间距、缩进、内边距等）

## 4. 数据流

### 4.1 静态渲染流程

```
Markdown文本
    ↓
[MarkdownParser] 解析
    ↓
AST节点列表
    ↓
[MarkdownRenderer] 渲染
    ↓
Widget树
    ↓
显示在屏幕
```

### 4.2 流式渲染流程

```
Stream<String>
    ↓
[StreamProcessor] 缓冲处理
    ↓
[IncrementalParser] 增量解析
    ↓
增量AST更新
    ↓
[MarkdownRenderer] 增量渲染
    ↓
Widget树更新
    ↓
平滑显示
```

## 5. 性能优化策略

### 5.1 解析优化
- ✅ 使用不可变数据结构（AST节点）
- 🔄 增量解析，只处理新增内容
- 🔄 并发解析（Isolate）
- 🔄 解析结果缓存

### 5.2 渲染优化
- 🔄 使用 `RepaintBoundary` 隔离
- 🔄 Widget 缓存和复用
- 🔄 虚拟滚动（长文档）
- 🔄 延迟加载图片

### 5.3 内存优化
- 🔄 及时释放不可见内容
- 🔄 图片内存管理
- 🔄 智能缓冲区大小
